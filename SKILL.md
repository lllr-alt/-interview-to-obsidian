# Interview to Obsidian

将面试题截图转化为结构化的 Obsidian 笔记，按主题自动分类到不同文件中。

## Vault 路径

Obsidian vault 路径：`/Users/lianxunming/Documents/Obsidian Vault/`

面试题统一存放在 vault 下的 `面试题/` 文件夹中。

## 工作流程

### 第一步：读取图片，提取面试题

仔细阅读用户上传的每张图片，逐题提取：
- 题目内容（完整保留原文，不要改写或缩略）
- 如果图片中包含答案，也一并提取
- 如果图片中没有答案，只提取题目，不要自行编造答案
- 如果图片中包含代码示例，也要提取并用代码块格式保留

**截断检测**：注意图片边缘处可能有被截断的内容——比如顶部出现了上一题答案的尾部，或底部题目/答案不完整。遇到这种情况，在对应题目后标注 *（图片截断，内容可能不完整）*，提醒用户检查。

### 第二步：分类并直接写入

提取完成后，根据题目内容判断所属领域，直接写入 Obsidian，不需要等用户确认分类。写入完成后在总结中列出分类结果，用户如果觉得不对可以提出调整。

这样做的原因是：减少交互轮次，用户看到结果后调整比事先确认更高效。

**常见分类**（不限于此，根据实际题目内容灵活创建新分类）：

| 分类 | 典型关键词/主题 |
|------|----------------|
| Agent | agent 架构、tool use、ReAct、planning、multi-agent |
| MCP | Model Context Protocol、MCP server、tool 注册 |
| RAG | 检索增强生成、embedding、向量数据库、chunk |
| Prompt Engineering | prompt 技巧、few-shot、CoT、system prompt |
| LLM 基础 | transformer、attention、tokenizer、微调、RLHF、预训练 |
| 多模态 | 图像理解、语音、视觉模型 |
| 机器学习 | 传统 ML、特征工程、XGBoost、时间序列、分类回归 |
| 工程实践 | 部署、API 设计、性能优化、评估、安全 |

分类原则：
- 一道题只归入一个最相关的类别，避免重复
- 如果某个类别只有 1 道题，考虑是否可以合并到相近的类别中（但如果该类别确实独立，保留也可以）
- 分类名用中文，文件名也用中文（如 `Agent.md`、`MCP.md`）

### 第三步：写入文件

**写入方式**：直接用 Write/Edit 工具将 markdown 文件写入 vault 目录。这比 obsidian-cli 更可靠，不依赖额外配置。Obsidian 会自动检测到文件变化并刷新。

**文件路径**：`/Users/lianxunming/Documents/Obsidian Vault/面试题/<分类名>.md`

**单个文件的结构**：

```markdown
---
tags:
  - 面试题
  - <分类名>
date: <今天的日期>
---

# <分类名> 面试题

## 题目一的完整内容？

答案内容（如果图片中有的话）。

如果图片中没有答案，写：*待补充*

## 题目二的完整内容？

答案内容...
```

**追加逻辑**：
- 如果文件已存在，先用 Read 工具读取现有内容，然后用 Edit 工具在末尾追加新题目（避免覆盖之前整理的题目）
- 如果文件不存在，用 Write 工具创建新文件
- 追加时不要修改 frontmatter 中的 date（保留首次创建日期）

**追加时的格式**：在已有内容末尾添加新题目，每题之间空一行：

```markdown

## 新题目内容？

答案...
```

### 第四步：总结

写入完成后，给用户一个简洁的总结：

```
已整理完成！共 X 道面试题，分布如下：

- 面试题/Agent.md — 3 道（新增 3 道）
- 面试题/MCP.md — 2 道（新增 2 道，已有 1 道）
- 面试题/RAG.md — 1 道（新增 1 道）

⚠️ 以下题目内容可能不完整（图片截断）：
- "给定一个时间序列..." — 答案被截断

可以在 Obsidian 中查看。还有其他图片需要整理吗？
```

## 注意事项

- 图片可能是手机截图、电脑截图、拍照等各种来源，文字排版可能不规整，要尽量准确提取
- 有些图片可能包含多道题目，要逐一识别
- 如果图片内容不是面试题（比如是文章、代码等），告知用户并询问是否仍要整理
- 如果同一张图片中混合了不同主题的题目，正常分类即可
- 保持题目原文的完整性，不要擅自修改措辞或简化题目
